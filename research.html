Research

1. Nuitka

Nuitka is a Python compiler that takes Python code and transpiles it to C. Then Nuitka takes the C code and compiles it into an executable. This is in contrast to CPython, the default implementation of Python. CPython simply executes the code as it interprets it. Nuitka has various advantages over both raw C and CPython. It is faster than CPython and has the speed of C. It also has ease of use of Python, making it easier for new students to contribute to the project. However, Nuitka does have the disadvantage of having a more involved setup.

2. DLL

A DLL (Dynamic Link Library) is a compiled module that contains functions for external programs to use. This is in contrast to a statically linked library, where all the compiled functions in the library are contained within the executable of the program using it.

When programming with a DLL over a statically linked library, it is easier to upgrade and fix bugs since one would just replace the DLL with a newer version. Additionally, using a DLL takes up less space. If two programs require the same DLL, only one DLL will actually be stored on the computer. In contrast, a statically linked library would have the library linked as part of the executable, so programs wouldn’t be able to share the library.

However, a DLL has a slower initialization, since the program has to load the DLL from the filesystem. Additionally, a missing DLL could lead to runtime errors in the program, but a statically linked library would guarantee that the library wouldn’t be missing.

3. MFC

Microsoft Foundation Class (MFC) is a C++ object-oriented wrapper library over Win32 for developing Windows desktop applications. It is used as an alternative to Win32 because Win32 API is very low-level. Compared to other higher-level frameworks, it is more clunky to use. Features that are easily available and easy to implement in higher-level frameworks like Qt or wxWidgets would take longer to implement in MFC. However, MFCs are essential to our work because they are backwards-compatible with previous versions of Windows. This is important in computers in governments and hospitals, because these computers are updated to new versions rarely for security reasons. MFCs are required by the NHS for any third-party programs.

4. Managing Github Repos

For our GitHub repository, every individual team is making their own modifications with MFCs. This results in a mess of branches on our GitHub repository. Instead of different teams going in different directions, teams must coordinate and build off each others work.
